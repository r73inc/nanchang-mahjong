## Open Bugs

### BUG-2D-01 — Claimed discard tile remains visible in the discard pool

**Status:** FIXED — PR #50
**Root cause:** `pung()`, `chow()`, and `kongFromDiscard()` in `packages/engine/src/engine.ts`
only patched the _claiming_ seat (adding the open meld) but never removed the claimed tile
from the _discarder's_ `discards` array. Because `toClientSnapshot` reads `seat.discards`
verbatim, the ghost tile appeared in every client snapshot until the hand ended.
**Fix:** Added `patchTwoSeats()` private helper to `GameEngine`. All three claim methods
now simultaneously patch the claiming seat (meld) and the discarder (slice last tile from
`discards`).
**Tests added:** 3 new engine regression tests in `engine.test.ts`:

- `pung removes the claimed tile from the discarder discards array`
- `chow removes the claimed tile from the discarder discards array`
- `kongFromDiscard removes the claimed tile from the discarder discards array`
  **Files changed:** `packages/engine/src/engine.ts`, `packages/engine/src/__tests__/engine.test.ts`

### BUG-2D-02 — Layout scaling is viewport-unaware; tiles overflow on smaller screens and after melds

**Status:** OPEN — deferred
**Symptoms (observed together):**

1. After completing a meld and then discarding, the viewer's hand is pushed off the
   bottom of the screen.
2. Expanding the browser window causes the left/right (rotated) opponent zones to shift
   and expand off-screen rather than staying anchored to their grid areas.
3. Left/right opponent tile rows appear to be scaled off the viewport _height_ rather
   than width, causing over-wide or over-narrow tile rows on non-square screens.
4. Must be compatible with mobile where the user cannot resize the browser.
   **Root cause (suspected):** The CSS Grid columns/rows are fixed percentages
   (`22% / 56% / 22%`) of the full-page container. The rotated side zones (left/right)
   use `transform: rotateZ(±90deg)` which swaps the element's visual width/height, but
   CSS still allocates a 22%-wide column for them in document flow. Tile size (`size="xs"`
   etc.) is a fixed-pixel prop, not viewport-relative, so as the grid shrinks (or grows
   after melds add height to the viewer zone) the tiles overflow their allocated area.
   **Where to look:**

- `apps/web/src/components/2d/GameTable2D.tsx` — grid column/row definitions and seat
  zone styles. Consider `vw`/`vh`/`dvh` units or a `ResizeObserver`-based approach
  so tile sizes track the actual rendered container.
- `apps/web/src/components/2d/layout-2d.ts` — `seatConfig()` assigns `containerTransform`
  and `role`. The `role` drives tile size; it may need to carry a dynamic size or the
  tile components need to derive size from CSS container queries.
- `apps/web/src/components/2d/OpponentHand2D.tsx` and `MahjongTile2D.tsx` — tile size
  props. Evaluate whether switching to `aspect-ratio` + `width: 100%` tiles inside
  a flex container would make them naturally fluid.
- `apps/web/src/components/2d/OpenMelds2D.tsx` — meld tiles push the viewer zone taller;
  the grid row height may need to shrink to compensate, or the outer wrapper should
  use `overflow: hidden` with `min-height: 0` so rows compress instead of overflow.

### BUG-2D-03 — Discarded tiles appear in each seat's zone rather than the centre of the table

**Status:** OPEN — deferred (improvement / design change)
**Symptom:** Each player's discarded tiles are rendered inside that player's own seat
zone (the 22% or 56% grid area), not in the shared centre area of the table. This
makes the table feel like four isolated panels rather than a communal playing surface.
**Expected behaviour:** All discards should land in the centre cell of the CSS Grid
(the `center` area, currently occupied only by `FeltSurface2D`). Each seat's discard
sub-region within the centre should be positioned near its respective edge (top discard
pile near the top edge, left near the left edge, etc.) so the spatial relationship is
preserved.
**Where to look:**

- `apps/web/src/components/2d/GameTable2D.tsx` — the `center` grid area currently
  contains only `<FeltSurface2D />`. The four `DiscardPool2D` instances would need to
  be moved into the centre cell and positioned absolutely (or via a nested grid) for
  each seat's region.
- `apps/web/src/components/2d/DiscardPool2D.tsx` — currently positioned relative to
  its seat zone container; will need absolute positioning and a `role`-based offset
  within the centre cell.
- `apps/web/src/components/2d/layout-2d.ts` — add discard-pool offset coordinates
  per seat role (top/right/bottom/left) within the centre area.
- Open melds should remain in the seat zones (per the bug report — "the melds are fine
  where they are").

## Key Learnings

- **Engine claim methods must patch two seats.** When a claim is resolved the claimed tile
  leaves the discarder's pile and enters the claimer's meld — the engine must update both
  seats atomically in one `withState` call using a two-seat patch. Patching only the
  claimer leaves a ghost entry in the discarder's `discards` array that propagates through
  every subsequent `toClientSnapshot` broadcast.
