01 · Locked decisions

What the prototype is committing to

Two interaction patterns were explored as Tweaks; both have been locked. Alternative variants have been removed from the code.

Tile discard

Tap a tile, then press Discard

Player taps the tile they want to discard — it lifts and a gold pill 出牌 / Discard ↑ appears above the hand. A second tap on Discard sends it. This double-confirm prevents misfires (the #1 complaint in family-mahjong forums) and reads cleanly to first-time players.

Alternates considered: drag-up-to-discard (felt natural but fired too easily during scroll on long hands); tap-tap-confirm (same number of taps but no explicit affordance).
Call prompts

Pung / Kong / Chow on the right rail

When another player's discard creates a possible call, the buttons stack on the right edge with a circular timer ring above them. The hand stays fully visible (no bottom-sheet occlusion), and the rail thumb-reaches naturally on a right-handed grip.

Alternates considered: bottom action sheet (covered the player's hand at the exact moment they need to see it); floating action buttons (no room for the urgency timer + context label).

07 · Motion

Restrained, with one star

Standard UI moves are 200–500ms on the Material standard curve. Reserve "emphasized" easing for high-stakes moments — the Jing reveal and the win banner. Everything else is fast and forgettable.

08 · Accessibility

What needs to be true before ship

This dark + gold palette is generally OK, but tile glyphs and small mono labels need attention. Mahjong is also a tile-recognition game — screen-reader users need a non-visual path.

✓
Touch targets ≥ 44 × 44 dp
Action buttons (Pung/Kong/Win) are 60 wide × 56 tall; bottom nav items are 44 tall; Discard pill is 36 tall but 88 wide — within Android Material guideline (48dp recommended, 44 minimum).
✓
Primary text on dark passes AA
Bone (#f5efdf) on Jade (#0d3b2e) = 13.7:1. Bone on Slate (#1f2937) = 14.9:1. Gold (#c9a961) on Jade = 5.8:1 — passes AA at all text sizes.
!
Tile glyph color contrast on dark face
The "Onyx" tile face (#1f2937 ink on #1f2937 face) breaks contrast. Bug, not a feature — must inverted-ink before ship. Theme.jsx already does this for face palettes < 0.4 luminance; verify on the Slate felt + Onyx face combo specifically.
!
Ink-dim (45%) caption is borderline
4.2:1 against Jade. Usable for ≥14px non-essential meta (date, "vs Mei, Wei, Lin"), but never use it on body or for critical info.
○
Screen-reader tile labels
Every <MahjongTile> must render an aria-label. Use the canonical name (e.g. "5 Bamboo", "East Wind", "Red Dragon"). Don't rely on the SVG glyph alone.
○
Focus order on Gameplay
Tab order: Menu → status bar → opponent nameplates (clockwise from top) → hand (left to right) → drawn tile → call rail → Discard. The current DOM order matches this; needs a keyboard pass to confirm. Add a visible focus ring (2px gold outline) to every interactive.
○
Reduced motion
Wrap the Jing reveal, last-discard pulse, and skeleton shimmer in @media (prefers-reduced-motion: reduce) stubs that swap to a static state. Tile discard animation stays (it conveys info).
○
Auto-discard timer announcement
When the 8-second timer crosses 3s, fire a polite ARIA live announcement: "3 seconds left to discard". Don't fire under 3s — too chatty.
○
Color-only state indicators
Ready/Not ready pill in the room lobby relies on color + label (good). Win/loss row stripes rely on color only — pair with a small ✓/✗ glyph or a leading "+/−" sign for color-blind players.
○
Form field labeling
Auth/Forgot-pw/Change-pw use eyebrow-style labels above the input. Make sure each is wired as a proper <label> with htmlFor (or wraps the input) — not just visual.
○
Language switcher
The LangToggle should set document.documentElement.lang when switched, so screen-readers swap voices for Chinese text.

10 · Open questions for dev

Things to confirm before implementation

These didn't fit cleanly into the design and want a conversation. Most have a default recommendation; mark them up if you disagree.

?
Auto-discard at timeout — what gets discarded?
Recommendation: the drawn tile (least surprising; matches most online clients). Alternative: the rightmost tile in hand.
?
Simultaneous claims (two players Pung the same discard)
Recommendation: priority is Win > Pung/Kong > Chow (standard rules). Surface a 600ms "two players called Pung" toast before resolution so the loser knows why.
?
Replay storage
The Replay screen assumes a flat move log on the server. Confirm format (we're proposing a JSON stream per match, ~38 entries × ~80 bytes = <4KB/game).
?
Share link routing
Replay share copies https://nanchang.mj/replay/{id}. Confirm: is the path public, family-friends-only, or requires auth?
?
Push notifications
Not designed yet. Likely needed for "your turn after reconnecting", "friend invited you", "rematch ready". Treat as a follow-up.
?
Landscape / tablet
Out of scope for v1. The current layout is portrait phone + centered column on desktop. If iPad-on-the-table emerges as a use case, the game screen specifically needs a wider rework.
